React 生命周期图示

React 的整个生命周期的流程图大致就是这样。
生命周期的流程可以描述为这几个阶段：

Initialization & mounting

  • getDefaultProps
  • getInitalState
  • componentWillMount
  • render
  • componentDidMount

State Change & Props Chnage

  • State Chagne
    • Upate State
    • ShouldComponentUpdate
    • ComponentWillUpdate

    • componentDidUpdate
  • Props Change
    • update props
    • componentWillRecivesProps
    • shouldComponentWillUpate
    • render
    • componentDidUpdate

Unmounting

  • unmounting
  • componentWillMount
1、Intitallzation

在上一章节的源码分析中，我们发现在 React.createClass中调用了getDefaultProps和getInitalState，在每次创建React组件时候都会进行调用且只调用一次。
mountComponent类负责对生命周期的initialization 和 mounting进行管理，即管理getDefaultProps 、getInitalState、componentWillMount、render、componentDidMount。
React生命周期其实也涉及到很多React DOM渲染相关的逻辑代码，由于有不同component的代码比较多，我只用典型的ReactCompositeComponent来分析。

代码中我们可以看到
mountComponent先做实例对象的props,state等初始化，然后调用performInitialMount初始化挂载，完成后调用componentDidMount。
代码中需要注意的是返回的markup为performInitialMountWithErrorHandling 和 performInitialMount生成。
performInitialMountWithErrorHandling 中也是运用了performInitialMount。
再来看performInitialMount的源码：

performInitialMount中先调用componentWillMount()，再将setState()产生的state改变进行state合并，然后调用_renderValidatedComponent()返回ReactElement，它会调用render()方法。然后由ReactElement创建ReactComponent。最后进行递归渲染。

由于递归的特性，父组件的 componentWillMount 一定在其子组件的 componentWillMount 之前调用，而父组件的 componentDidMount 肯定在其子组件的 componentDidMount 之后调用。

2 Update

updateComponent 负责管理生命周期中的 componentWillReceiveProps、shouldComponentUpdate、componentWillUpdate、render 和 componentDidUpdate。
我们其实是用setState来更新组件，也就是通过updateComponent来对组件进行更新。

代码中调用 shouldComponentUpdate 判断是否需要进行组件更新，如果存在 componentWillUpdate，则执行。
updateComponent 本质上也是通过 递归渲染内容的哦。

当渲染完成之后，若存在 componentDidUpdate就进行触发。

3 Unmounting

更新组件时，如果不满足DOM diff条件，会先unmountComponent, 然后再mountComponent，
unmountComponent通过调用componentWillUnmount
并且递归调用unmountComponent来销毁组件。
